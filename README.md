# Spark — A Post-Match Concept for Dating Apps

A post-match interaction concept designed to reduce the pressure of starting conversations on dating apps.


**Prototype:**

## The Problem

Introverted users on dating apps often struggle after matching. Instead of starting conversations, they hesitate, overthink, or avoid initiating altogether. This leads to missed connections and lower engagement despite successful matches.

There's a clear gap in the post-match experience — where emotional comfort and interaction design need to work together.



## The Concept

Instead of opening to a blank chat, **Spark** intercepts the post-match moment with a lightweight, low-pressure flow:

1. **Private check-in** — Both users answer a few quick questions independently, drawn from their profiles and shared interests. Neither can see the other's answers yet.
2. **Shared reveal** — The app surfaces what they have in common and where they differ, giving both people something natural to react to and not a prompt, but a real overlap.
3. **Quick Draw** — An optional collaborative drawing round where both users draw the same thing simultaneously on separate canvases. No score, no winner. Just a shared imperfect moment.
4. **Natural chat entry** — The conversation opens with something already happened between them. The blank page anxiety is gone.

> The goal: make the first interaction feel less performative and more human.



## Design Decisions

**Why not prompt suggestions?**
Prompts are generic. The check-in questions in Spark are generated from each user's actual profile - so they change person to person and feel personal, not automated.

**Why drawing?**
A collaborative activity removes the performance pressure of "saying the right thing." Both people are equally bad at drawing a confused cat on a phone. That shared imperfection is the point — it creates a moment, not a competition.

**Why no score?**
Scoring introduces anxiety. The round ends on a timer and transitions automatically. The result is something they made together, not a judgement of either person.

**Design system**
Built within Hinge's visual language — Cormorant Garamond for headings, Bitter for body, and a palette that extends Hinge's cream, black, and purple accent system. The feature feels native, not bolted on.



## Screens

| Screen | Description |
|--------|-------------|
| Match + Check-in | Post-match moment with private binary questions |
| Shared Reveal | Matched and different answers surfaced with context |
| Quick Draw | Simultaneous collaborative drawing canvas with timer |
| Draw Result | Shared sketch revealed with a natural transition to chat |
| Chat | Conversation opens with a shared reference point already established |



**Brief requirements met:**
- Small number of screens executed at high quality over a broad shallow flow
- Originality — avoided generic prompt suggestions
- Clickable Figma prototype with minimal annotations on the board
- Designed within an existing app's (Hinge) design system
